# 用户操作手册

## 1. 使用前准备

### 1.1 启动系统

先启动后端和前端：

```bash
uvicorn src.main:app --host 0.0.0.0 --port 8000 --reload
streamlit run src/ui/app.py
```

默认访问地址：

- 前端：`http://127.0.0.1:8501`
- 后端健康检查：`http://127.0.0.1:8000/health`

### 1.2 当前系统说明

当前版本没有登录模块，打开前端即可进入系统。

首页中的核心指标目前是演示用静态数据，不代表实时数据库统计。

## 2. 页面总览

当前前端包含 9 个页面：

1. 首页概览
2. 报告上传
3. 议题识别
4. 差距分析
5. 人工复核
6. P0条款复核
7. 对标分析
8. 审计日志
9. 规则配置

其中“对标分析”页面当前仍为规划中展示页，不提供实际分析能力。

## 3. 报告上传

### 3.1 支持格式

上传页面支持：

- PDF
- Word
- Excel

### 3.2 操作步骤

1. 打开“报告上传”页面
2. 点击文件选择框
3. 选择待分析文件
4. 点击“开始处理文件”
5. 等待后端完成解析与入库

### 3.3 处理完成后可查看内容

处理成功后，页面会展示：

- 文件名
- 文件大小
- 文本长度
- 分块数量
- 修复后文本预览

如果处理失败，页面会显示后端返回的错误信息。

## 4. 议题识别

### 4.1 进入方式

完成报告上传后，可以：

- 在上传页点击“进入信息抽取”
- 或直接打开“议题识别”页面

### 4.2 当前页面行为

当前版本不会要求你再点一次“开始识别”。页面检测到已有 `corpus_result` 后，会自动发起议题识别与知识库检索。

### 4.3 结果展示

识别完成后，页面会显示：

- 识别议题数量
- 检索到的标准条文数量
- 检索到的同行案例数量
- 按高/中/低置信度分组的议题列表
- 每个议题对应的证据原文
- 相关标准条文摘要

## 5. 差距分析与优化建议

### 5.1 差距分析

在“差距分析”页面，点击“获取差距分析”后，系统会基于已识别议题生成：

- 整体评估
- 重大差距
- 轻微差距
- 符合要求项
- 同行对比信息

### 5.2 优化建议

在同一页面点击“生成优化建议”后，系统会生成：

- 优先行动项
- 分议题建议列表
- 一段可编辑的完整建议文本

### 5.3 导出

当前页面支持下载：

- 优化建议文本（`.txt`）

确认进入人工复核后，还可以在下一页导出完整 JSON。

## 6. 人工复核

### 6.1 进入条件

人工复核页依赖上一页生成的建议内容。如果没有待复核内容，系统会提示先回到“差距分析”页生成建议。

### 6.2 可执行操作

对每条建议，你可以：

- 查看原建议
- 直接编辑建议文字
- 标记复核状态：待复核、通过、不采纳、已修改
- 填写复核备注

### 6.3 页面输出

复核页会自动统计：

- 已通过数量
- 已修改数量
- 不采纳数量
- 待复核数量
- 整体复核进度

### 6.4 导出

当前页面支持导出：

- 已复核建议文本（`.txt`）
- 含复核状态与备注的完整报告（`.json`）

## 7. P0条款复核

### 7.1 进入条件

P0条款复核页使用 E4 pending-review 数据。进入页面前需要先启动后端，并执行过：

```bash
python scripts/seed_stage_e4_pending_review.py
```

当前页面接入的是 Stage E accepted artifacts：

- 143 条 current disclosure assessments
- 143 条 Advisor coverage
- 复核记录表 `p0_review_decisions`

### 7.2 页面状态

页面会显示：

- `review_status=pending`
- `final_evaluation_status=pending_human_evaluation`
- Advisor 状态为 `AI-assisted recommendation pending human review`

这些状态表示当前数据等待人工评测，不代表人工评测已经完成。

### 7.3 可执行操作

在页面中可以：

1. 选择 E4 run。
2. 按 AI verdict、review status、GRI standard、disclosure 或关键词筛选条款。
3. 查看条款级 evidence、PDF 页码、报告页码、requirement checks 和 Advisor coverage。
4. 填写 human verdict、evidence page check、requirement gap check、error type、correction note 和 reviewer。
5. 保存复核记录到 SQLite。

保存复核记录只写入 `p0_review_decisions`，不会修改 AI 原始 assessment 或 Advisor 原始建议。

### 7.4 导出

页面支持下载：

- assessment CSV/JSON
- Advisor CSV/JSON
- review decisions CSV/JSON

导出文件保留 AI 原始字段、证据页码、requirement IDs、manual review reason、Advisor recommendation text 和人工复核字段。导出文件不包含阶段 F 评测指标。

### 7.5 F workbook 回流

人工填写 F workbook 后，可通过脚本导入复核结果：

```bash
python scripts/import_f_human_review_results.py path/to/review_rows.json
```

导入脚本会校验 `manifest_item_id`、`assessment_id` 或 `advisor_item_id` 是否能匹配 E4 数据。未知 row key 会被拒绝并返回错误清单。

## 8. 对标分析

“对标分析”页面当前仅用于展示规划方案，不提供真实计算结果。

你现在可以从页面中看到：

- 计划中的输入方式
- 预期分析维度
- 规划中的输出图表
- 当前开发进度

## 9. 审计日志

### 9.1 日志查询

在“审计日志”页面可以按以下条件查询：

- 操作类型
- 操作人
- 时间范围
- 返回数量

### 9.2 结果内容

查询结果会展示：

- 时间戳
- 操作类型
- 操作人
- 状态
- 日志哈希值
- 操作详情

每条记录还支持单独做完整性校验。

### 9.3 统计与导出

当前页面支持：

- 生成操作类型统计
- 导出 JSON
- 导出 CSV
- 手动计算任意文本的 SHA-256 哈希值

## 10. 规则配置

### 10.1 可配置内容

规则配置页当前提供 5 类能力：

1. 实质性议题规则配置
2. ESG 披露标准配置
3. 匹配策略配置
4. 抽取字段配置
5. 规则备份与恢复

### 10.2 典型操作

你可以直接在页面中：

- 编辑或删除已有议题
- 添加新议题
- 添加新披露标准
- 调整关键词/正则/语义匹配权重
- 备份当前规则为 JSON
- 从备份文件恢复规则

所有规则修改都会写入审计日志。

## 11. 常见注意事项

### 11.1 AI 输出使用限制

系统生成的议题识别、差距分析和优化建议都只是辅助内容，不能直接作为正式披露结论。

### 11.2 数据存储位置

当前系统会把运行数据写入本地：

- `data/chroma_db/`
- `data/raw_corpus/`
- `data/sqlite_db/`
- `data/export_results/`

### 11.3 对外发送说明

本地文件会保存在本机，但文本片段会发送到你配置的大模型和嵌入服务。

### 11.4 当前未实现项

以下功能当前还没有真正打通：

- 多企业对标分析
- 对话窗口（检索问答）
- 披露标准版本差异分析
