# P0/P1/P2 需求对齐表

## 使用口径

P0 是“单报告 + GRI 披露证据核验”的验证样例，目标是证明公开报告证据、GRI requirement、LLM 辅助判断、人工复核和审计留痕可以形成可追溯闭环。P0 不代表完整 ESG 智能分析平台。

## 对齐表

| 需求来源或访谈方向 | P0 落点 | P1/P2 保留 | 当前状态 |
|---|---|---|---|
| ESG 专员需要快速核对单份报告披露情况 | 远景能源 2024 中文 ESG 报告的 GRI 披露证据核验 | 多报告批处理、跨年度趋势 | P0 已进入 E3 准备 |
| GRI 披露条款逐项核验 | requirement checklist、requirement_checks、五分类 verdict、人工复核 | 完整多语种标准库、更多标准框架互转 | P0 已完成 E2.1 基础契约 |
| 报告证据可追溯 | evidence_kind、source_page、report_page_label、source_text、chunk_id | PDF 可视化标注、证据截图导出 | P0 已完成基础字段，UI 展示在 E4 |
| 人工复核与误差复盘 | manual_review_input、manual_review_result、smoke review、阶段门 | 多人协同复核、权限、复核工作流统计 | P0 脚本层可用，E4 做页面 |
| 下一期披露改进建议 | Advisor 基于 requirement 缺口输出报告内容建议 | 企业内部管理诊断、任务派发、整改追踪 | P0 只做证据边界内建议 |
| 友商或竞品对标 | 不进入 P0 当前结论 | P1：多企业 benchmark、行业分位、披露深度对比 | 规划中 |
| 客户、融资银行、资本市场关注点 | 不作为 P0 评分依据 | P1/P2：利益相关方视角评分和材料生成 | 规划中 |
| 第三方 ESG 数据库接入 | 不进入 P0 | P2：外部评级、新闻、处罚、数据库交叉验证 | 未实现 |
| 供应链风险监测 | 不进入 P0 | P2：供应商画像、风险信号、供应链舆情 | 未实现 |
| Claw 舆情或实时监测 | 不进入 P0 | P2：舆情监测、预警、事件归因 | 未实现 |
| 登录、多用户、组织协作 | 不进入 E3 | P1/P2：账户、权限、协作复核 | 未实现 |
| Dashboard 和管理驾驶舱 | E4 仅做 P0 Streamlit 条款级工作台 | P1/P2：管理看板、指标追踪、对标驾驶舱 | P0 不接入外部静态 dashboard |

## 写作和展示限制

- 不把 P1/P2 能力写成 P0 已实现。
- 不把公开报告缺失直接写成企业内部不存在。
- 不把 `readiness_2026` 或 `watchlist_2027` 纳入 2024 current-gap 披露统计。
- 不把 E3 写成“118 条全量分析”；E3 是 2024 当前披露范围分批运行。
