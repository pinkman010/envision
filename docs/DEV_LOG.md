# 开发日志 (DEV_LOG)

## 项目概述
- **项目名称**: ESG披露与沟通智能分析系统
- **版本**: v1.0.0
- **开发周期**: 2025年

## 核心功能模块

### 1. 语料处理模块 (CorpusAgent)
- **功能**: PDF/Word文档解析、文本分块、向量化存储
- **技术栈**: PyPDF2, python-docx, ChromaDB, Ollama嵌入模型
- **状态**: ✅ 已完成

### 2. 信息抽取模块 (ExtractAgent)
- **功能**: 按预设字段清单提取事实内容、字符级锚点绑定、相似度校验
- **技术栈**: LLM调用、Levenshtein相似度算法
- **状态**: ✅ 已完成

### 3. 合规提示模块 (ComplianceAgent)
- **功能**: 按ISSB/SASB/HKEX标准标注风险点
- **技术栈**: 白盒规则匹配、LLM辅助提示
- **状态**: ✅ 已完成

### 4. 内容生成模块 (ContentAgent)
- **功能**: 固定模板填充、文本润色
- **技术栈**: Jinja2模板、LLM生成
- **状态**: ✅ 已完成

### 5. 审计日志模块
- **功能**: 全链路留痕、SHA-256哈希防篡改、查询导出
- **技术栈**: SQLite、哈希算法
- **状态**: ✅ 已完成

## 关键技术决策

### 1. 向量数据库选择
- **选择**: ChromaDB
- **理由**: 轻量级、本地部署、支持持久化

### 2. 嵌入模型选择
- **选择**: Ollama + nomic-embed-text
- **理由**: 本地部署、数据不出境、免费使用

### 3. 大模型选择
- **选择**: Moonshot AI (kimi-k2.5)
- **理由**: 中文理解能力强、支持长文本、性价比高

### 4. 数据库选择
- **选择**: SQLite
- **理由**: 轻量级、无需额外服务、适合单人/小团队使用

## 合规设计原则

1. **AI仅辅助，人工最终决策**
   - 所有AI输出仅为提示，不构成任何披露建议
   - 最终决策权100%在企业ESG团队手里

2. **白盒可解释**
   - 所有专业判断由纯代码实现，100%可审计
   - AI仅做事实提取，不做专业判断

3. **全链路留痕**
   - 所有操作记录审计日志
   - 日志带SHA-256哈希，防篡改
