# 部署说明文档

## 1. 适用范围

本文档对应当前 P0 交付状态，适用于本地开发、课程演示和小范围内部复核。

当前可直接使用的产品能力是：

- 固定报告：远景能源 2024 中文 ESG 报告。
- 固定范围：143 条 current disclosure assessment units。
- 固定工作台：Streamlit 条款复核页。
- 固定状态：`review_status=pending`、`final_evaluation_status=pending_human_evaluation`。

本文档不描述多企业对标、登录、多用户权限、任务队列、生产部署和任意报告自动完整分析。这些能力属于后续工程化范围。

## 2. 系统组成

当前部署包含两部分：

| 组件 | 说明 |
|---|---|
| FastAPI 后端 | 提供 `/health`、`/api/v1/p0-review/*` 等接口 |
| Streamlit 前端 | 提供首页、报告上传、条款复核、审计日志页面 |

P0 条款复核工作台使用已接受的 Stage E artifacts，不会在页面打开时重新调用 143 条 LLM 分析。

数据流如下：

```text
Stage E accepted artifacts
  -> scripts/seed_stage_e4_pending_review.py
  -> SQLite p0_review_* tables
  -> FastAPI /api/v1/p0-review
  -> Streamlit 条款复核页面
```

## 3. 环境要求

### 3.1 软件要求

- Python：`>=3.10`
- 推荐使用项目专用 Conda 或虚拟环境
- 依赖声明来源：`pyproject.toml`

安装依赖：

```powershell
python -m pip install --upgrade pip
python -m pip install -e ".[dev]"
```

### 3.2 项目内 runtime bridge

如环境目录无写权限，项目支持从 `vendor/python_runtime/` 加载 API 启动依赖。该目录为本地运行补丁，不进入版本库。

缺失时可重建：

```powershell
python -m pip install --target vendor/python_runtime "anyio>=4.0.0" "uvicorn[standard]>=0.24.0"
```

启动 API 时优先使用：

```powershell
python scripts/run_api.py
```

## 5. 初始化 P0 工作台数据

首次使用条款复核工作台前，执行：

```powershell
python scripts/seed_stage_e4_pending_review.py
```

脚本默认读取：

- 143 条 accepted current assessments
- 143 条 Advisor coverage
- traceability cleanup maps
- F advisor review coverage sheet

写入 SQLite 后，后端 API 和 Streamlit 页面即可读取数据。

## 6. 启动服务

### 6.1 启动后端

```powershell
python scripts/run_api.py
```

默认访问：

```text
http://127.0.0.1:8000
```

检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8000/health
```

### 6.2 启动前端

```powershell
streamlit run src/ui/app.py
```

默认访问：

```text
http://127.0.0.1:8501
```

如端口被占用，可指定新端口：

```powershell
streamlit run src/ui/app.py --server.port 8521
```

## 7. 启动后检查

建议按以下顺序检查：

1. 访问 `/health`，确认后端返回 `healthy`。
2. 访问 `/api/docs`，确认 FastAPI 文档可打开。
3. 打开 Streamlit 首页，确认导航包含“条款复核”。
4. 在条款复核页选择 run：`20260630T113930Z_e3_final_current_effective_set_accepted`。
5. 确认页面显示 143 条 assessments 和 143 条 Advisor coverage。
6. 打开任意条款详情，确认可查看 requirement checks、evidence、页码和 source text。
7. 保存一条测试人工复核记录。
8. 导出 assessment、Advisor 或 review decisions 的 CSV/JSON。

页面和导出中应继续显示：

```text
review_status=pending
final_evaluation_status=pending_human_evaluation
AI-assisted recommendation pending human review
```

## 9. 常见问题

### 9.1 后端启动时报缺少 `uvicorn`

先重建项目内 runtime bridge：

```powershell
python -m pip install --target vendor/python_runtime "anyio>=4.0.0" "uvicorn[standard]>=0.24.0"
python scripts/run_api.py
```

### 9.2 条款复核页面没有数据

先写入 E4 pending-review 数据：

```powershell
python scripts/seed_stage_e4_pending_review.py
```

### 9.3 页面数量不是 143

检查 seed 脚本读取的 Stage E accepted artifacts 是否存在，并查看脚本输出的 summary 文件。

### 9.4 是否需要重新调用 LLM

使用 E4 条款复核工作台不需要重新调用 LLM。只有重新执行 Stage E/E3/E3.5 或统一 Advisor 生成脚本时才会调用外部模型。
