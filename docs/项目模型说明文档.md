# 项目模型说明文档

## 一、系统架构

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                        接入层                            │
│              Streamlit UI + FastAPI                     │
├─────────────────────────────────────────────────────────┤
│                        应用层                            │
│                    1+5 Agent架构                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐│
│  │CorpusAgent│  │ExtractAgent│  │ComplianceAgent│  │ContentAgent││
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘│
├─────────────────────────────────────────────────────────┤
│                        核心能力层                        │

├─────────────────────────────────────────────────────────┤

│              Chroma向量库  SQLite数据库                  │

```

### 1.2 Agent架构

#### 1.2.1 BaseAgent（基类）
- **职责**: 定义Agent通用接口和状态管理
- **方法**: `run()`, `_execute()`, `reset()`
- **状态**: IDLE, RUNNING, SUCCESS, FAILED, PENDING

#### 1.2.2 CorpusAgent（语料处理）
- **职责**: PDF/Word解析、文本分块、向量化存储
- **输入**: 文件路径
- **输出**: 语料ID、分块信息

#### 1.2.3 ExtractAgent（信息抽取）
- **职责**: 按预设字段提取事实内容、字符级锚点绑定
- **输入**: 语料处理结果
- **输出**: 结构化提取结果

#### 1.2.4 ComplianceAgent（合规提示）
- **职责**: 按ESG标准标注风险点
- **输入**: 信息抽取结果
- **输出**: 合规提示列表

#### 1.2.5 ContentAgent（内容生成）
- **职责**: 固定模板填充、文本润色
- **输入**: 人工确认数据
- **输出**: 标准化披露文本

## 五、算法模型

### 5.1 相似度算法

#### 5.1.1 Levenshtein距离
- **用途**: 计算字符串相似度
- **公式**: similarity = 1 - (distance / max_length)
- **阈值**: 0.98（可配置）

#### 5.1.2 向量相似度
- **用途**: 语义检索
- **公式**: cosine_similarity = dot(a, b) / (norm(a) * norm(b))
- **阈值**: 0.7（可配置）

### 5.2 规则匹配算法

#### 5.2.1 关键词匹配
- **权重**: 0.3
- **方式**: 精确匹配 + 模糊匹配

#### 5.2.2 正则匹配
- **权重**: 0.5
- **方式**: 数值提取模式

## 六、配置模型

### 6.1 环境变量配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| LLM_API_KEY | 大模型API密钥 | 必填 |
| LLM_BASE_URL | 大模型API地址 | 必填 |
| SIMILARITY_THRESHOLD | 相似度阈值 | 0.98 |

| CHUNK_OVERLAP | 分块重叠大小 | 100 |

- **位置**: config/rule_templates/
- **格式**: JSON
- **热更新**: 支持（通过规则配置页面）
