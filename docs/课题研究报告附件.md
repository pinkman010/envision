# 课题研究报告附件

## 1. 研究背景

### 1.1 行业痛点

新能源行业ESG披露面临三大挑战：

1. **披露质量参差不齐**：行业缺乏统一的披露标准和最佳实践
2. **信息提取效率低**：人工阅读数百页ESG报告耗时费力
3. **合规风险难把控**：多标准（ISSB/SASB/HKEX）要求难以同时满足

### 1.2 现有解决方案局限

| 方案类型 | 代表产品 | 局限性 |
|----------|----------|--------|
| 通用大模型 | ChatGPT、Claude | 幻觉严重、无法溯源、无行业定制 |
| 传统NLP工具 | 命名实体识别 | 准确率低、需要大量标注数据 |
| 人工咨询 | 四大会计师事务所 | 成本高、周期长、难以规模化 |

## 2. 研究目标

构建**「规则驱动为主、AI辅助为辅」**的ESG智能分析系统，实现：

- ✅ 高准确：抽取准确率>[目标值]
- ✅ 强合规：字符级溯源、审计留痕
- ✅ 可落地：MVP阶段即可企业试用
- ✅ 低成本：无需大量标注数据

## 3. 研究方法

### 3.1 技术路线

采用「1+5轻量化Agent混合架构」：

1. **MasterAgent**：固定流程调度，0 AI参与
2. **CorpusAgent**：语料处理，AI仅修复异常（10%参与度）
3. **ExtractAgent**：事实抽取，AI仅做提取（100%参与度但受限）
4. **ComplianceAgent**：合规提示，规则兜底
5. **ContentAgent**：内容生成，模板填充

### 3.2 合规机制设计

- **字符级双向溯源**：AI输出绑定原文精确位置
- **三重幻觉锁死**：输入锁死→输出锁死→校验锁死
- **白盒规则兜底**：专业判断100%纯代码实现
- **人工最终决策**：所有披露内容需人工复核

## 4. 研究创新点

### 4.1 理论创新

1. **「约束即能力」框架**：通过严格约束AI权限，反而提升系统可靠性
2. **「规则-AI-人工」三层架构**：明确定义每层职责边界
3. **ESG领域适配**：针对新能源行业15个核心议题定制规则

### 4.2 实践创新

1. **固定Prompt设计**：每个Agent仅1条Prompt，零调试成本
2. **字符级溯源**：实现AI输出的100%可追溯
3. **零代码配置**：业务规则外置，企业可自主调整

## 5. 研究局限性

### 5.1 当前限制

1. **语种限制**：目前主要支持中英文ESG报告
2. **格式限制**：扫描版PDF识别准确率有待提升
3. **规模限制**：MVP阶段单报告处理，批量处理能力待完善

### 5.2 未来方向

1. 扩展至其他高碳行业（化工、钢铁、建筑）
2. 引入多模态能力（ESG数据可视化图表解析）
3. 构建行业ESG知识图谱

## 6. 伦理与合规声明

### 6.1 数据使用

- 所有训练语料来源于公开披露的企业ESG报告
- 已进行脱敏处理，删除敏感商业数据
- 仅用于学术研究，不做商业用途

### 6.2 AI伦理

- 系统设计遵循「人类最终决策」原则
- 明确告知用户AI输出的局限性
- 建立人工复核和申诉机制
