# 部署说明文档

## 1. 适用范围

本文档对应当前仓库实现，适用于本地开发、课程演示和小范围内部试运行。

当前系统包含两部分：

- FastAPI 后端：提供语料处理、议题识别、差距分析、优化建议 API
- Streamlit 前端：提供上传、分析、复核、审计和规则配置界面

## 2. 环境要求

### 2.1 硬件建议

- CPU：4 核及以上
- 内存：8GB 及以上
- 磁盘：至少 20GB 可用空间

### 2.2 软件要求

- 操作系统：Windows、macOS 或 Linux
- Python：`3.10+`
- 推荐环境：使用 `environment.yml` 创建 `Python 3.13` Conda 环境

### 2.3 网络要求

当前实现默认依赖外部 API：

- `LLM_BASE_URL` 指向的 OpenAI 兼容大模型服务
- `https://api.siliconflow.cn/v1` 嵌入服务

如果你改为自建兼容服务，需要同步修改 `.env` 中对应地址和密钥。

## 3. 关键目录

```text
envision/
├── src/                     # 后端、前端与 Agent 实现
├── templates/               # Prompt、规则、导出模板

│   ├── raw_corpus/          # 解析后的原始/修复文本

│   └── export_results/      # 导出结果
├── logs/                    # 运行日志

├── .env.example             # 环境变量模板
├── environment.yml          # Conda 环境定义
└── pyproject.toml           # Python 依赖与项目元数据
```

## 4. 安装步骤

### 4.1 获取代码

```bash
git clone <repository_url>
cd envision
```

### 4.2 创建运行环境

推荐使用 Conda：

```bash
conda env create -f environment.yml
conda activate envision
```

也可以直接使用现有 Python 环境：

```bash
python -m pip install --upgrade pip
python -m pip install -e ".[dev]"
```

### 4.3 配置环境变量

复制模板文件：

```bash
cp .env.example .env
```

Windows PowerShell：

```powershell
Copy-Item .env.example .env
```

当前代码对大部分配置项要求较严格，至少需要确认以下字段已经填写：

- 项目基础：`PROJECT_NAME`、`PROJECT_DESCRIPTION`、`VERSION`、`API_PREFIX`
- 服务地址：`ENVIRONMENT`、`HOST`、`PORT`、`API_BASE_URL`、`ALLOWED_ORIGINS`
- 大模型：`LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL`
- 嵌入模型：`SILICONFLOW_API_KEY`、`EMBEDDING_MODEL`
- 存储：`SQLITE_DB_NAME`、`CHROMA_DB_PERSIST_DIR`
- 文件与分块：`MAX_FILE_SIZE`、`CHUNK_SIZE`、`CHUNK_OVERLAP`

建议直接参考 `.env.example` 完整填写。

### 4.4 导入知识库（可选但推荐）

如果需要标准条文检索和同行案例参考，建议先导入知识库。

导入标准条款：

```bash
python scripts/import_standards.py
```

扫描同行报告目录：

```bash
python scripts/import_peer_reports.py --dry-run
```

实际导入同行报告：

```bash
python scripts/import_peer_reports.py
```

说明：

- 两个脚本都会写入 `data/chroma_db/`
- 如果集合里已有数据，脚本会提示是否清空后重建

## 5. 启动服务

### 5.1 启动后端

```bash
uvicorn src.main:app --host 0.0.0.0 --port 8000 --reload
```

### 5.2 启动前端

```bash
streamlit run src/ui/app.py
```

### 5.3 访问地址

- 前端：`http://127.0.0.1:8501`
- 健康检查：`http://127.0.0.1:8000/health`
- Swagger：`http://127.0.0.1:8000/api/docs`
- ReDoc：`http://127.0.0.1:8000/api/redoc`

## 6. 启动后检查

服务首次启动时，程序会自动创建以下目录：

- `data/chroma_db/`
- `data/sqlite_db/`
- `data/raw_corpus/`
- `data/export_results/`
- `logs/app/`
- `logs/api/`

建议按下面顺序检查：

1. 访问 `/health`，确认后端返回 `healthy`
2. 打开 `/api/docs`，确认接口文档可访问
3. 打开前端首页，确认左侧导航和页面切换正常
4. 上传一份小体积 PDF 或 DOCX，确认能完成 `Corpus -> Retrieval -> Analyst -> Advisor` 流程

## 7. 常见问题

### 7.1 启动时报缺少环境变量

原因：

- `.env` 未创建
- `.env` 中有字段遗漏
- 当前终端不在仓库根目录

处理方式：

- 确认根目录存在 `.env`
- 对照 `.env.example` 补齐所有必填项
- 从仓库根目录重新启动

### 7.2 大模型调用失败

优先检查：

- `LLM_API_KEY` 是否有效
- `LLM_BASE_URL` 是否可访问
- `LLM_MODEL` 是否为目标服务支持的模型名

### 7.3 向量库初始化或嵌入失败

优先检查：

- `SILICONFLOW_API_KEY` 是否有效
- `EMBEDDING_MODEL` 是否为 `BAAI/bge-m3`
- 当前网络是否可访问 `api.siliconflow.cn`
- `data/chroma_db/` 是否有写权限

### 7.4 前端无法连接后端

优先检查：

- 后端是否已经启动在 `PORT`
- `.env` 中 `API_BASE_URL` 是否与实际后端地址一致
- `.env` 中 `ALLOWED_ORIGINS` 是否包含前端地址

### 7.5 上传文件失败

优先检查：

- 文件格式是否为 PDF、Word 或 Excel
- 文件大小是否超过 `MAX_FILE_SIZE`
- 文件是否包含无法解析的扫描内容或损坏页

## 8. 备份与升级

### 8.1 建议备份内容

- `.env`
- `data/`
- `templates/rule_templates/`

### 8.2 升级后建议动作

1. 重新安装依赖
2. 检查 `.env.example` 是否新增配置项
3. 验证 `/health`
4. 用一份样例报告跑通主流程

## 9. 安全说明

- 当前默认会把原始文本、修复文本和向量数据保存在本地磁盘
- 当前默认会把文本片段发送到你配置的 LLM 与嵌入服务
- 不要把真实 `.env` 提交到 Git
- 对外演示前，建议清理测试语料和历史审计日志
