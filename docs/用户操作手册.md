# 用户操作手册

## 1. 系统访问

### 1.1 启动方式

**方式一：Docker一键部署（推荐）**
```bash
# Linux/Mac
chmod +x deploy/quick_start.sh
./deploy/quick_start.sh

# Windows
双击运行 deploy/quick_start.bat
```

**方式二：源码启动**
```bash
# 启动后端
python -m src.main

# 启动前端（新终端）
streamlit run src/ui/app.py
```

### 1.2 访问地址

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端界面 | http://localhost:8501 | 主要操作界面 |
| API文档 | http://localhost:8000/docs | 后端接口调试 |

## 2. 核心功能操作

### 2.1 ESG语料管理

**上传ESG报告**
1. 进入「ESG语料管理中心」
2. 点击侧边栏「上传ESG报告」
3. 选择PDF或Word文件（最大50MB）
4. 点击「开始处理语料」
5. 等待处理完成，查看元数据和原文预览

**语料版本管理**
- 系统支持按「企业+年份+版本号」分层存储
- [待实现] 查看历史版本对比功能

### 2.2 实质性议题分析

**单报告分析流程**
1. 确保已完成语料上传
2. 进入「实质性议题分析中心」
3. 系统自动执行：
   - 信息抽取（AI提取+字符级锚点）
   - 相似度校验（防止幻觉）
   - 议题匹配（白盒规则）
4. 查看抽取结果和置信度
5. 对低置信度结果进行人工确认

**查看溯源**
- 点击任意抽取字段的「查看原文」
- 系统自动定位到原文字符位置
- 高亮显示对应文本段落

### 2.3 合规提示检查

1. 进入「披露优化与策略助手」
2. 系统自动：
   - 扫描缺失披露项
   - 对比ISSB/SASB/HKEX标准
   - 标记合规风险点
3. 查看风险提示和建议
4. 导出合规检查报告

### 2.4 导出分析报告

1. 完成分析后，点击「导出报告」
2. 选择导出格式：
   - Excel：议题分析、对标数据
   - Word：披露优化方案
3. 下载到本地

## 3. 人工复核流程

### 3.1 复核入口

进入「人工复核中心」，查看待复核项目列表。

### 3.2 复核操作

1. 查看AI抽取结果和原文对照
2. 确认或修改抽取内容
3. 签署复核意见
4. 提交复核结果（带数字签名）

### 3.3 复核留痕

- 所有复核操作自动记录审计日志
- 支持复核历史查询
- 不可篡改（SHA-256哈希）

## 4. 审计日志查询

1. 进入「审计日志中心」
2. 设置查询条件：
   - 时间范围
   - 操作类型
   - 用户ID
3. 查看操作详情
4. 导出审计报告（CSV/Excel）

## 5. 常见问题

**Q: 上传文件大小限制？**  
A: 单个文件最大50MB，支持PDF、DOCX、DOC格式。

**Q: 处理超时怎么办？**  
A: 处理时间取决于报告页数、复杂度、网络环境和大模型API响应速度。如长时间无响应，请检查网络连接或联系管理员。

**Q: AI抽取结果不准确？**  
A: 系统会标记低置信度结果，请进入人工复核中心确认修改。

**Q: 如何更新业务规则？**  
A: 联系管理员修改 `config/rule_templates/` 目录下的JSON配置文件。
