# 部署说明文档

## 一、环境要求

### 1. 硬件要求
- **CPU**: 4核及以上
- **内存**: 8GB及以上
- **磁盘**: 50GB可用空间

### 2. 软件要求
- **操作系统**: Linux/macOS/Windows (WSL2)
- **Python**: 3.11+
- **Docker**: 20.10+ (可选)
- **Ollama**: 最新版 (本地嵌入模型)

## 二、部署方式

### 方式一：Docker一键部署（推荐）

#### 1. 克隆代码
```bash
git clone <repository_url>
cd esg-system
```

#### 2. 配置环境变量
```bash
cp .env.example .env
# 编辑.env文件，填写实际配置
```

#### 3. 启动服务
```bash
docker-compose up -d
```

#### 4. 访问服务
- 前端UI: http://localhost:8501
- 后端API: http://localhost:8000
- API文档: http://localhost:8000/api/docs

### 方式二：手动部署

#### 1. 安装依赖
```bash
pip install -r requirements.txt
```

#### 2. 启动Ollama服务
```bash
ollama serve
# 在另一个终端
ollama pull nomic-embed-text
```

#### 3. 配置环境变量
```bash
cp .env.example .env
# 编辑.env文件
```

#### 4. 启动后端API
```bash
python -m uvicorn src.main:app --host 0.0.0.0 --port 8000
```

#### 5. 启动前端UI
```bash
streamlit run src/ui/app.py
```

## 三、环境变量配置

### 必需配置
```bash
# 大模型API
LLM_API_KEY=your-api-key-here
LLM_BASE_URL=https://api.moonshot.cn/v1

# Ollama嵌入模型
OLLAMA_BASE_URL=http://localhost:11434
```

### 可选配置
```bash
# 相似度阈值（默认0.98）
SIMILARITY_THRESHOLD=0.98

# 最大文件大小（默认200MB）
MAX_FILE_SIZE=209715200

# 日志级别（默认INFO）
LOG_LEVEL=INFO
```

## 五、常见问题

### Q: Docker部署失败？
A: 检查：
1. Docker服务是否启动
2. 端口是否被占用
3. .env文件是否配置正确

### Q: Ollama连接失败？
A: 检查：
1. Ollama服务是否启动
2. OLLAMA_BASE_URL配置是否正确
3. 防火墙是否放行端口

### Q: LLM调用失败？
A: 检查：
1. LLM_API_KEY是否正确
2. 网络连接是否正常
3. LLM服务是否可用

## 六、升级维护

### 1. 备份数据
```bash
cp -r data data_backup_$(date +%Y%m%d)
```

### 2. 更新代码
```bash
git pull origin main
```

### 3. 重启服务
```bash
docker-compose down
docker-compose up -d
```
