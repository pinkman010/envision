# 项目模型说明文档

## 1. 架构概述

### 1.1 核心设计理念

本项目采用**「规则驱动为主、AI辅助为辅」**的强合规架构设计：

- **白盒规则兜底**：所有专业判断、合规判定由纯代码硬规则完成
- **AI仅做事实提取**：大模型仅负责从原文复制内容，无创作、无判断权限
- **人工最终决策**：所有对外披露内容必须经人工复核确认

### 1.2 「1+5」Agent混合架构

```
┌─────────────────────────────────────────────────────────────┐
│                     MasterAgent (总控)                      │
│              0 AI参与 · 固定流程状态机                       │
└──────────────┬──────────────────────────────────────────────┘
               │
    ┌──────────┼──────────┬──────────┬──────────┐
    ▼          ▼          ▼          ▼          ▼
┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐
│Corpus │ │Extract│ │Compli-│ │Content│ │Audit  │
│Agent  │ │Agent  │ │ance   │ │Agent  │ │(工具) │
│10%AI  │ │100%AI │ │Agent  │ │Agent  │ │       │
└───────┘ └───────┘ └───────┘ └───────┘ └───────┘
```

| Agent | AI参与度 | 核心职责 |
|-------|----------|----------|
| MasterAgent | 0% | 固定流程调度、状态跟踪、异常上报 |
| CorpusAgent | 10% | PDF/Word解析、文本分块、AI仅修复乱码 |
| ExtractAgent | 100% | 事实提取+字符级锚点，无判断权限 |
| ComplianceAgent | 低 | 合规风险高亮标注，无决策权限 |
| ContentAgent | 中 | 模板填充，无自主创作 |

## 2. 核心合规机制

### 2.1 字符级双向溯源

所有AI抽取的内容必须绑定原文精确字符位置：

```python
{
    "field_name": "scope1_emission",
    "extracted_content": "12,500吨CO2e",
    "char_start": 15234,
    "char_end": 15247,
    "page_number": 45,
    "source_text": "2023年度范围一排放量为12,500吨CO2e，"
}
```

### 2.2 三重幻觉锁死

| 锁死层级 | 机制 | 实现方式 |
|----------|------|----------|
| 输入锁死 | 温度=0 | LLM参数固定，输出确定性 |
| 输出锁死 | JSON Schema | 强制结构化输出，拒绝自由文本 |
| 校验锁死 | 相似度阈值 | Levenshtein算法，相似度<0.98直接拦截 |

### 2.3 全链路审计

- 所有操作永久留痕（操作者、时间戳、输入、输出）
- SHA-256哈希防篡改
- 支持审计日志导出和哈希校验

### 3.2 披露标准映射

支持3套国际披露标准：

- **ISSB**: IFRS S1/S2 可持续披露准则
- **SASB**: 可持续会计准则委员会标准
- **HKEX**: 香港交易所ESG报告指引

## 4. 技术实现

### 4.1 核心约束

- **固定Prompt**：每个Agent仅1条Prompt，无需调试
- **固定字段**：信息抽取字段清单100%写死
- **固定流程**：MasterAgent仅支持3条预设流程
- **硬规则校验**：所有校验纯代码实现，无AI参与

### 4.2 异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| AI输出格式错误 | 拦截 + 记录审计日志 |
| 相似度校验失败 | 标记人工复核 + 记录置信度 |
| 规则匹配失败 | 走兜底规则 + 上报异常 |

## 5. 扩展性设计

- **业务规则外置**：`config/`目录零代码配置
- **多模型适配**：预留`src/adapters/`目录
- **多用户权限**：预留`src/auth/`目录
