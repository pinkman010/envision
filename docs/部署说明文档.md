# 部署说明文档

## 1. 系统要求

### 1.1 最低配置

| 资源 | 要求 | 说明 |
|------|------|------|
| CPU | 2核 | 支持AVX指令集 |
| 内存 | 4GB | 推荐8GB以上 |
| 存储 | 20GB | SSD推荐 |
| 网络 | 可访问互联网 | 用于调用大模型API |

### 1.2 推荐配置

| 资源 | 要求 | 说明 |
|------|------|------|
| CPU | 4核+ | 并发处理更流畅 |
| 内存 | 8GB+ | 向量库加载更快速 |
| 存储 | 50GB+ SSD | 支持更多语料存储 |
| GPU | 可选 | 本地嵌入模型加速 |

### 1.3 软件依赖

- Docker 20.10+
- Docker Compose 2.0+
- 或 Python 3.10+（源码部署）

## 2. 部署方式

### 方式一：Docker一键部署（推荐）

**适用场景**：生产环境、快速验证

**步骤**：

1. **获取代码**
```bash
git clone <repository-url>
cd envision
```

2. **配置环境变量**
```bash
# 复制模板文件
cp .env.example .env

# 编辑.env，填入必要的API密钥
# 必须配置：LLM_API_KEY
# 可选配置：LLM_BASE_URL, LLM_MODEL等
```

3. **执行一键部署脚本**
```bash
# Linux/Mac
chmod +x deploy/quick_start.sh
./deploy/quick_start.sh

# Windows（管理员权限运行CMD或PowerShell）
deploy\quick_start.bat
```

4. **验证部署**
```bash
# 检查容器状态
docker-compose -f deploy/docker-compose.yml ps

# 查看日志
docker-compose -f deploy/docker-compose.yml logs -f
```

5. **访问系统**
- 前端：http://localhost:8501
- API文档：http://localhost:8000/docs

### 方式二：源码部署

**适用场景**：开发环境、自定义修改

**步骤**：

1. **创建虚拟环境**
```bash
python -m venv venv

# Linux/Mac
source venv/bin/activate

# Windows
venv\Scripts\activate
```

2. **安装依赖**
```bash
pip install -r requirements.txt
```

3. **配置环境变量**
```bash
cp .env.example .env
# 编辑.env文件
```

4. **创建数据目录**
```bash
mkdir -p data/chroma_db
mkdir -p data/sqlite_db
mkdir -p data/raw_corpus/versioned
mkdir -p data/raw_corpus/unversioned
mkdir -p data/export_results
mkdir -p logs
```

5. **启动后端**
```bash
python -m src.main
```

6. **启动前端（新终端）**
```bash
streamlit run src/ui/app.py
```

## 4. 常见问题排查

### 4.1 容器启动失败

**症状**：`docker-compose up` 报错退出

**排查步骤**：
1. 检查Docker服务是否运行
2. 检查端口8000、8501是否被占用
3. 查看容器日志：`docker-compose logs api`

### 4.2 无法连接大模型API

**症状**：语料处理提示"API连接失败"

**排查步骤**：
1. 检查 `.env` 中 `LLM_API_KEY` 是否配置
2. 检查网络是否能访问 `LLM_BASE_URL`
3. 检查API密钥余额是否充足

### 4.3 前端无法连接后端

**症状**：页面提示"网络请求失败"

**排查步骤**：
1. 确认后端服务已启动
2. 检查防火墙设置
3. 检查 `.env` 中 `ALLOWED_ORIGINS` 是否包含前端地址

## 5. 数据备份

### 5.1 需要备份的目录

```bash
# 数据库
data/sqlite_db/

# 向量库
data/chroma_db/

# 原始语料
data/raw_corpus/

# 导出结果
data/export_results/

# 日志（可选）
logs/
```

### 5.2 备份脚本示例

```bash
#!/bin/bash
BACKUP_DIR="/backup/envision/$(date +%Y%m%d)"
mkdir -p $BACKUP_DIR

tar -czf $BACKUP_DIR/data.tar.gz data/
tar -czf $BACKUP_DIR/config.tar.gz config/

echo "备份完成: $BACKUP_DIR"
```

## 6. 升级维护

### 6.1 系统升级

```bash
# 拉取最新代码
git pull origin main

# 重新构建镜像
docker-compose -f deploy/docker-compose.yml build

# 重启服务
docker-compose -f deploy/docker-compose.yml up -d
```

### 6.2 日志清理

```bash
# 清理7天前的日志
find logs/ -name "*.log" -mtime +7 -delete
```

## 7. 安全建议

1. **API密钥保护**：切勿将真实密钥提交到Git仓库
2. **访问控制**：生产环境建议配置反向代理和HTTPS
3. **数据隔离**：不同企业的数据应使用独立的存储卷
4. **定期备份**：建议每日自动备份关键数据
